🖼️ Adding Images & GIFs

Learn how to add images and GIFs to your case studies using local storage.

Media Folder Structure

All project images are stored in the public/media/ folder:

public/media/ ├── projects/ ← Project thumbnails & images (served) │ ├── bbc/ │ │ └── thumbnail.png │ ├── doula/ │ │ └── thumbnail.png │ └── [other projects]/ media/ ├── company_logo/ ← Related Experience logos (imported via JS) │ ├── sirelo_logo.jpg │ └── [other logos] └── logo.png ← Site logo

Thumbnail Image (Required)

Each project needs one thumbnail image used for both the work list view AND the project hero section.

Property Value Notes
Location public/media/projects/[slug]/ Slug must match project filename
Location media/projects/[slug]/ Slug must match project filename
Filename thumbnail.png Always use this exact name
Minimum Width 640px Display width is 320px (2x for retina)
Height Auto Keeps original aspect ratio, no cropping
Aspect Ratio Any 16:9, 4:3, 1:1, 3:4 all work fine
Format PNG, JPG, WebP PNG recommended for quality

Key Insight: One thumbnail serves BOTH the work list AND the hero section. No separate hero image needed.

Supported Formats

Format Extension Supported
Images .png, .jpg, .jpeg, .webp ✅ Yes
Animations .gif ✅ Yes
SVG .svg ✅ Yes

Article Images (Optional)

For images within your case study content, place them in the same project folder:

public/media/projects/my-project/
├── thumbnail.png          ← Required (used in list + hero)
├── screenshot1.png       ← Optional (for article content)
└── screenshot2.jpg      ← Optional (for article content)

Reference in markdown:

![Image description](/media/projects/my-project/screenshot1.png)

Note: Article images use relative paths from content/projects/. Keep all project images in the same named folder as your markdown file.

Adding GIFs

GIFs work exactly like images - just use the .gif extension:

![Watch the interaction](media/projects/my-project/interaction-demo.gif)

Tip: GIFs are great for showing:

  • Interactive prototypes
  • User flows
  • Animation details
  • Before/after comparisons

Quick Reference

  • Thumbnail path - /media/projects/[slug]/thumbnail.png in frontmatter
  • Article images - Same folder, reference with /media/projects/[slug]/filename.png
  • Minimum width - 640px (display is 320px)
  • Height - Auto (no cropping needed)
  • Aspect ratio - Any works
  • GIFs - Same as images, use .gif extension
  • Storage - Always use public/media/projects/[slug]/

📝 Adding New Work Items

Step-by-step guide to adding a new project case study to your portfolio.

Overview

Projects are stored as markdown files in content/projects/. Each file represents one case study with:

  • Frontmatter - Metadata at the top (title, client, date, etc.)
  • Content - Your case study written in markdown
  • Thumbnail Image - Stored locally in media/projects/[slug]/thumbnail.png

Step 1: Create the File & Folder

Create a new markdown file in content/projects/ and a folder in public/media/projects/:

content/projects/my-project.md
public/media/projects/my-project/thumbnail.png

Filename rules:

  • Use lowercase letters
  • Use hyphens (-) instead of spaces
  • Keep it short but descriptive
  • Example: sirelo-shipping-calculator.md

Folder Structure: The project folder name must match the markdown filename (without extension).

Step 2: Add Frontmatter

Every project needs frontmatter at the top of the file. Copy this template:

---
title: "Your Project Title"
client: "Company Name"
date: "2024-01-01"
description: "A short description for project cards."
tags: ["Tag1", "Tag2"]
thumbnail: "/media/projects/my-project/thumbnail.png"
---

# Your Project Title

Your case study content starts here...

Required Fields

  • title - Project title (appears as page title)
  • client - Company/client name (used to link to experience cards)
  • date - Date in YYYY-MM-DD format
  • description - Short summary shown on project cards

Optional Fields

  • tags - Array of tags for filtering
  • thumbnail - Local path to thumbnail image
  • hidden - Set to true to hide from grids

Step 3: Add Thumbnail Image

Place your thumbnail image in public/media/projects/[slug]/thumbnail.png

Image Specifications

Property Value Notes
Location public/media/projects/[slug]/ Must match project slug
Filename thumbnail.png Always "thumbnail"
Width 640px minimum Fixed display width is 320px
Height Auto (keep aspect ratio) No cropping needed
Aspect Ratio Any 16:9, 4:3, 1:1, 3:4 all work
Formats PNG, JPG, WebP PNG recommended

Important: The same thumbnail image is used for both the work list view AND the project hero section. No separate hero image needed.

Step 4: Write Your Content

After the frontmatter (the --- line), write your case study content using markdown.

See Content Formatting Guide for details.

Step 5: Link to Experience (Optional)

If you want a "Related Experience" card to appear on your project page, the client name must match an entry in the experience mapping.

Currently Supported Clients

  • Sirelo - Shows Sirelo experience card
  • CoorpID (ING Bank) - Shows CoorpID experience card
  • Sagefund - Shows Sagefund experience card

To add more, edit src/app/projects/[slug]/page.tsx and add to clientExperienceMap.

Step 6: Test Your Project

Build the project: Run npm run build to verify everything compiles correctly.

Note: Hidden projects are accessible via direct URL but won't appear in the project grid.

Quick Checklist

  • ✅ Created file in content/projects/
  • ✅ Created folder in public/media/projects/[slug]/
  • ✅ Added thumbnail image (640px+ width, auto height)
  • ✅ Added all required frontmatter fields
  • ✅ Wrote case study content
  • ✅ Tested with npm run build
